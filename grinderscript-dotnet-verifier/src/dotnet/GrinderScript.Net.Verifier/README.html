Verifying the GrinderScript.Net installation

This folder contains some example load tests that you can use to verify that The Grinder can run GrinderScript.Net load tests.

You verify the installation by following these five steps.
  1. Start the Grinder Console
  2. Start one or more Grinder Agents
  3. Set the Grinder Console root directory to this verifier folder. (From the Distribute -> "Set directory.." menu)
  4. Select the Verification.grinder.properties file as the active properties. (From the Script tab, mark the file, right click and Select properties)
  5. Start a new run, either from the Action -> "Start process" menu or from the upper leftmost button.
Hopefully the run will start as expected, and you'll see the TPS ticking of and the Graph and Results tabs will contain information about the run.

If the run doesn't start, or if it shutsdown without any data beeing collected by the Grinder Console, you'll have to troubleshoot the problem by looking in the agent console window and/or the agent logs.

The property grinder.duration=30000, so the verification will run for 30 seconds before it shuts down again.

Have fun!
